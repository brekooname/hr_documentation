In this module we will tackle the database structure of the HR Gateway, which table is use in which page, etc.

Accountability Setting

Accountability Setting is the table that stores the Accountability Setting Linkings from the Linking Sessions

									Table Name:
									accblty_conv
								
									Structure:
									
ac_id::int(10) (Primary Key, Auto Incremented)

									
ac_link_id::int(10) (Index Key, From Linking Master)

									
ac_com_id::varchar(50) (Index Key, An md5 Version of the ac_link_id, URL to access session)

									
ac_focus::tinyint(4) (From Linking Focus)

									
ac_skill::varchar(250) (Behavior/Skill)

									
ac_when_use::varchar(250) (When we use the Skill)

									
ac_how_use::varchar(250) (How we use the Skill)

									
ac_why_use::varchar(250) (Why we use the Skill)

									
ac_expectations::varchar(250) (My Expectations)

									
ac_expectation_date::date (Date Expectation was Set)

									
ac_comments::mediumtext (Comments)

									
ac_feedback::mediumtext (Feedback)

									
created_at::datetime (Date the data is inserted)

									
updated_at::datetime (Date the data is updated)

									
ac_status::tinyint(4) (Status of the Linking Session)

Additional Linkees

Additional Linkees is the table that stores an additional Link between 2 employees that is not an Immediate Superior or a Manager of another employee. The Linker can make a new Linking Session with the Linkee.

									Table Name:
									adtl_linkees
								
									Structure:
									
adtl_id::int(10) (Primary Key, Auto Incremented)

									
adtl_linker::smallint(5) (The Linker Employee, from Employee Information)

									
adtl_linkee::smallint(5) (The Linkee Employee, from Employee Information)

									
adtl_row::tinyint(3) (Row number)

									
adtl_added_by::smallint(5) (The user who added the linking)

									
adtl_date_added::datetime (Date the data is inserted)

									
adtl_status::tinyint(4) (Status of the linking)
Comments
  • Plan to change some of the data types that is being used.

Disciplinary Action (Infraction)

Disciplinary Action (Infraction) is the table that stores employee's sanctions.

									Table Name:
									da_infraction
								
									Structure:
									
id::int(10) (Primary Key, Auto Incremented)

									
employee_id::int(11) (Index Key, From Employee Information)

									
title::varchar(200) (Title of the sanction, eCode Type)

									
date::date (Date of the Incident)

									
file::text (URL of the File Uploaded)

									
infraction_type::enum('NTE', 'NOD') (Type of Infraction: NTE or NOD)

									
filed_by::int(11) (The user who created the data, from Employee Information)

									
reason::text (The explanation of the employee that is being sanctioned)

									
status::int(11) (Status of the Infraction)

									
affixed_name::varchar(200) (Signature of the employee that is being sanctioned)

									
acknowledged_date::datetime (Date of acknowledgement)

									
hr_notes::text (Notes from HR Department)

									
supervisor_id::int(11) (the supervisor of the employee, From Employee Information)

									
manager_id::int(11) (the manager of the employee, From Employee Information)

									
recommend_date::datetime (Date the supervisor/manager recommended the infraction)

									
recommend_note::text (Note from the supervisor after recommendation)

									
recommend_note_manager::text (Note from the manager after recommendation)

									
approved_date::datetime (Date the manager approved the infraction)

									
approved_note::text (Note from the manager after approving)

									
approved_status::enum('NONE', 'RECOMMENDED', 'APPROVED') (infraction approval status)

									
remarks::enum('PENDING', 'COMPLETED') (infraction remarks, if NOD is already filed)

									
created_at::datetime (Date the data is inserted)

									
updated_at::datetime (Date the data is updated)

									
deleted_at::datetime (Date the data is deleted)
Comments
  • Status numeric value
    • 0 - Not Acknowledged
