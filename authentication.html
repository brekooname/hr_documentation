Authentication

Overview

This module will tackle the Employee Login and Forgot Password functionalities.

Employee Login

This is the page where the employee will be authenticating his/her credentials to use the Gateway

									Location:
									http://dir.elink.corp/login
								
									Inputs Needed:
									
Username (Employees Email)

									
Password

								
									Tables Affected:
									employee_info
								
									Models Used:
									root/app/User.php
								
									Controllers Used:
									root/app/Http/Controllers/Auth/LoginController.php
									root/app/Http/Controllers/EmployeeInfoController.php
								
									Views Used:
									root/resources/views/auth/login.blade.php
									root/resources/views/auth/styles.php
								
									Other Directory Used:
									root/app/Repositories/AuthRepository.php
								
									Functions Used:
									
EmployeeInfoController::login

									
AuthRepository::login

									
AuthRepository::authFields
After the employee inputs his/her credentials
  • EmployeeInfoController::login will received the posted credentials and pass it to AuthRepository::login
  • AuthRepository::login will determine which app it is being accessed either a Browser or Lightweight Directory Access Protocol (LDAP) then passes it to AuthRepository::authFields
  • AuthRepository::authFields will then check if the credentials are correct and return to back to the login page or return to a new page
Comments
  • Plan to remove Lightweight Directory Access Protocol (LDAP)

Forgot Password

This is the page where the employee will be typing his/her Email Address when the employee forgets his/her Credentials

									Location:
									http://dir.elink.corp/password
								
									Inputs Needed:
									
Username (Employees Email)

								
									Tables Affected:
									employee_info
								
									Models Used:
									root/app/User.php
								
									Controllers Used:
									root/app/Http/Controllers/PasswordResetController.php
								
									Views Used:
									root/resources/views/passwords/reset.blade.php
									root/resources/views/passwords/success.blade.php
									root/resources/views/auth/styles.php
								
									Other Directory Used:
									root/app/Mail/ResetPasswordMail.php
									root/resources/views/passwords/email_token.blade.php
								
									Functions Used:
									
PasswordResetController::reset

									
PasswordResetController::confirmReset
After the employee inputs his/her Email Address
  • PasswordResetController::reset will received the Email Address, if the email existed, the system will send an Email to that Email Address with a link that the Employee has to click to confirm the Password Reset.
  • It will redirect to the PasswordResetController::confirmReset function which will change the password to the users birthday.
Comments
  • Sample Password: December251992
  • First Letter of the Month always in Capital Letter
  • If the birthdate day is less that 10, add Zero for the first number. eg: January011990
