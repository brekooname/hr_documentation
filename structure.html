Overview

This is the overview of the files inside the system.

app

The directory contains numerous subfolders, which will be addressed later. The files on inside the app directory consists of all the php models (since the laravel version does not provide a model directory) and the Global.php settings.

Console

The directory contains the Kernel.php file use for task scheduling, it contains the function to add Leave Credits to all employees every month base on their ranks.

Exceptions

The directory contains the application's exception handler Handler.php.

(This file was not touched)

Helpers

The directory contains all the helper functions use for developing.

(Note that if you create a new custom helper file you need to run a certain code before it becomes usable)

									composer dump-autoload

HTTP

The directory contains the folders for Controllers and Middleware which will be addressed later. It also has its own Kernel.php file which provides a structured process for converting a Request into a Response.

(this file was not touched)

Controllers

The directory contains all of the controllers used for developing, all of the functions that are use to return a specific result.

Middleware

The directory contains all of the middlewares used for developing. It provides a convenient mechanism for inspecting and filtering HTTP requests entering your application.

Jobs

The directory contains the UpdateEmployeeCredits.php of which the function is used in the main Console/Kernel.php for updating Employees Leave Credits monthly.

Mail

The directory contains the files for Mailing.

Providers

The directory contains the files for providing service to the application. It also contains the RepositoryServiceProvider.php where all the Repositories are registered.

Repositories

The directory contains the files for authenticating the employee when logging in and the employee information.

(If new repository is added, refer to providers)

bootstrap

The directory contains the app.php file which bootstraps the framework.

config

The directory contains all of the application's configuration files.

database

The directory contains your database migrations, model factories, and seeds.

public

The directory contains the index.php file, which is the entry point for all requests entering your application and configures autoloading. This contains all the assets used, css, javascript, images, fonts, and other attachments.

resources

The directory contains the views as well as the raw, un-compiled assets such as LESS, SASS, or JavaScript. This directory also houses all of the language files.

routes

The directory contains all of the route definitions for your application.

Do not forget, when adding functions in the controller you need to add it on the web.php as well

storage

The directory contains the compiled Blade templates, file based sessions, file caches, and other files generated by the framework. The employees profile images are stored in app/images.

tests

The directory contains your automated tests.

(files not touched)

vendor

The directory contains the Composer dependencies.

(files not touched)
